ローカルでの動作確認からnpmパッケージ公開までの一連のフローを実行するスキル。

## 手順

### 1. ビルドとローカル動作確認
1. `npm run build` — ビルド
2. `npm link` — ローカルのビルド結果をグローバルに反映
3. ユーザーに動作確認を促す
4. ユーザーからOKが出るまで公開に進まない

### 2. READMEの確認
- 今回の変更内容にREADMEの更新が必要か確認する
- 必要であればユーザーに伝え、更新してから次に進む

### 3. バージョンの決定
- `package.json` の現在のバージョンを読み取る
- ユーザーに変更内容の種別を確認し、適切なバージョンを提案する
  - バグ修正・小さな改善 → パッチ (例: 2.3.0 → 2.3.1)
  - 新機能追加 → マイナー (例: 2.3.1 → 2.4.0)
  - 破壊的変更 → メジャー (例: 2.4.0 → 3.0.0)
- **必ずユーザーの承認を得てからバージョンを変更する**

### 4. 公開
1. `package.json` のバージョンを更新
2. `npm run build` — バージョン更新後に再ビルド
3. `npm publish` — npmレジストリに公開

### 5. クリーンアップ
1. `npm unlink -g claude-memex` — リンク解除
2. `npm install -g claude-memex` — レジストリ版をグローバルにインストール

## ルール
- バージョンは直前のバージョンから連番で上げる。飛ばさない
- バージョンをユーザーに確認せずに勝手に決めない
- 動作確認のOKを得ずに公開しない
